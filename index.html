Wholesale Book Store Management System
Course: CSI204 - Software Engineering Workshop #1

📖 ภาพรวมโครงการ | Project Overview

ในปัจจุบันพฤติกรรมการซื้อสินค้าของผู้บริโภคได้เปลี่ยนแปลงไปสู่ช่องทางออนไลน์มากขึ้น ร้านขายหนังสือแบบดั้งเดิมจึงจำเป็นต้องปรับตัวเพื่อตอบสนองความต้องการของลูกค้าที่ต้องการความสะดวกสบายในการเลือกซื้อสินค้าจากทุกที่ทุกเวลา โครงงานนี้จึงพัฒนาระบบร้านขายหนังสือออนไลน์ขึ้น เพื่อเป็นช่องทางจำหน่ายหนังสือแบบปกอ่อนผ่านระบบอินเทอร์เน็ต โดยมีการจัดส่งสินค้าผ่านบริษัทขนส่งและมีระบบติดตามสถานะคำสั่งซื้อให้แก่ลูกค้าด้วย

🎯 วัตถุประสงค์โครงการ | Objectives

1. รองรับการใช้งานของผู้เยี่ยมชมและลูกค้าสมาชิก

เพื่อพัฒนาระบบร้านหนังสือออนไลน์ที่รองรับผู้ใช้งาน 2 ประเภท ได้แก่ ผู้เยี่ยมชมทั่วไปและลูกค้าที่สมัครสมาชิก โดยผู้เยี่ยมชมสามารถสมัครบัญชี เข้าชมหน้ารวมสินค้า ค้นหาสินค้า และดูรายละเอียดสินค้าได้ ส่วนลูกค้าที่เข้าสู่ระบบแล้ว สามารถใช้งานฟังก์ชันสั่งซื้อสินค้าได้ครบถ้วน

2. อำนวยความสะดวกในการค้นหาและเลือกซื้อหนังสือ

เพื่อให้ลูกค้าสามารถเข้าชมรายการหนังสือทั้งหมด ค้นหาหนังสือที่ต้องการ ดูรายละเอียดสินค้า ตรวจสอบจำนวนสินค้าคงเหลือ และรับคำแนะนำหมวดหนังสือที่เกี่ยวข้อง เพื่อช่วยประกอบการตัดสินใจก่อนสั่งซื้อ

3. จัดการบัญชีผู้ใช้ รายการโปรด และข้อมูลจัดส่ง

เพื่อให้ลูกค้าสมาชิกสามารถเข้าสู่ระบบ จัดการข้อมูลโปรไฟล์ของตนเอง เช่น ชื่อผู้ใช้ รหัสผ่าน เพิ่มหรือแก้ไขที่อยู่จัดส่ง บันทึกหนังสือลงรายการโปรด และดูประวัติคำสั่งซื้อของตนเองได้อย่างสะดวก

4. รองรับกระบวนการสั่งซื้อและชำระเงิน

เพื่อพัฒนาระบบที่ช่วยให้ลูกค้าสามารถเพิ่มสินค้าลงตะกร้า จัดการสินค้าในตะกร้า เลือกรูปแบบการจัดส่ง สั่งซื้อสินค้า ชำระเงินผ่านเลขบัญชี และแนบหลักฐานการชำระเงินเข้าสู่ระบบได้

5. คำนวณยอดชำระและตรวจสอบสต็อกสินค้า

เพื่อให้ระบบสามารถเปรียบเทียบจำนวนสินค้าในสต็อกกับจำนวนสินค้าที่ลูกค้าสั่งซื้อ แสดงจำนวนสินค้าคงเหลือ และคำนวณราคาสินค้ารวมกับค่าจัดส่ง เพื่อแสดงเป็นยอดสุทธิที่ลูกค้าต้องชำระ

6. จัดการสถานะการจองสินค้าและการตัดสต็อก

เพื่อให้ระบบสามารถจัดการสถานะสินค้าได้อย่างถูกต้อง โดยเมื่อลูกค้าโอนเงินและแนบสลิป ระบบจะเปลี่ยนสถานะสินค้าจาก “ว่าง” เป็น “จองสินค้า” และหากพนักงานยืนยันคำสั่งซื้อแล้ว ระบบจะตัดสต็อกอัตโนมัติ โดยเปลี่ยนสถานะจาก “จองสินค้า” เป็น “ขายแล้ว” แต่หากพบว่าสลิปปลอมหรือมีการยกเลิกคำสั่งซื้อ ระบบจะคืนสต็อกกลับเป็นสถานะ “ว่าง”

7. ติดตามคำสั่งซื้อและสถานะการจัดส่ง

เพื่อให้ลูกค้าสามารถติดตามคำสั่งซื้อและสถานะการจัดส่งสินค้าได้ โดยระบบจะแสดงข้อความแจ้งเตือน เช่น “คำสั่งซื้อของคุณอยู่ระหว่างการตรวจสอบ” รวมถึงเปิดให้ลูกค้ายืนยันการได้รับสินค้าเมื่อได้รับสินค้าเรียบร้อยแล้ว

8. รองรับการติดต่อสอบถามและแจ้งปัญหา

เพื่อให้ลูกค้าสามารถติดต่อสอบถามหรือแจ้งปัญหาเกี่ยวกับการสั่งซื้อ การชำระเงิน การจัดส่ง หรือการใช้งานระบบ ผ่านช่องทางอีเมลได้

9. สนับสนุนการทำงานของพนักงาน

เพื่อให้พนักงานสามารถรับการแจ้งเตือนเมื่อมีคำสั่งซื้อใหม่ ตรวจสอบรายละเอียดคำสั่งซื้อ ตรวจสอบหลักฐานการชำระเงิน ยืนยันคำสั่งซื้อ อัปเดตสถานะคำสั่งซื้อ จัดเตรียมสินค้า อัปเดตสถานะการจัดส่ง และดูประวัติคำสั่งซื้อของลูกค้าได้

10. จัดการข้อมูลหลังบ้านสำหรับแอดมิน

เพื่อให้แอดมินสามารถจัดการระบบหลังบ้านได้อย่างครบถ้วน ได้แก่ การเพิ่มหรือลดพนักงาน การเพิ่มสินค้าเข้าสู่หน้าร้าน การจัดการจำนวนสินค้าในสต็อก การอัปเดตจำนวนสินค้าเพิ่มหรือลด และการตรวจสอบสินค้าที่ใกล้หมด

11. ตรวจสอบการอนุมัติคำสั่งซื้อ

เพื่อให้แอดมินสามารถตรวจสอบรายละเอียดการอนุมัติคำสั่งซื้อได้ โดยสามารถดูได้ว่าพนักงานคนใดเป็นผู้รับผิดชอบหรือดำเนินการกับคำสั่งซื้อนั้น เพื่อเพิ่มความถูกต้องและความโปร่งใสในการทำงาน

12. สรุปข้อมูลยอดขายและการจัดส่งผ่านกราฟ

เพื่อให้แอดมินสามารถดูข้อมูลสรุปของร้านผ่านกราฟต่าง ๆ ได้แก่ ยอดคำสั่งซื้อรายอาทิตย์ รายเดือน และรายปีในรูปแบบกราฟแท่ง ยอดขายรายอาทิตย์ รายเดือน และรายปีในรูปแบบกราฟเส้น สถานะการจัดส่งรายวัน รายอาทิตย์ และรายเดือนในรูปแบบกราฟแท่ง รวมถึงหมวดหมู่หนังสือที่ขายได้รายอาทิตย์ รายเดือน และรายปีในรูปแบบกราฟวงกลม

13. แจ้งเตือนสินค้าคงเหลือต่ำ

เพื่อให้ระบบสามารถแจ้งเตือนแอดมินเมื่อสินค้าแต่ละรายการมีจำนวนคงเหลือน้อยกว่า 100 เล่ม ช่วยให้สามารถวางแผนเติมสินค้าได้ทันเวลาและลดปัญหาสินค้าหมดสต็อก

📊 ขอบเขตงาน | Scope

  1. ระบบสมัครสมาชิก เข้าสู่ระบบ และจัดการบัญชีผู้ใช้งาน
  2. ระบบเข้าชมหน้ารวมสินค้า ค้นหาสินค้า และดูรายละเอียดหนังสือ
  3. ระบบบันทึกรายการโปรดและแนะนำหมวดหนังสือที่เกี่ยวข้อง
  4. ระบบตะกร้าสินค้า การจัดการจำนวนสินค้า และการสั่งซื้อสินค้า
  5. ระบบเพิ่มและจัดการที่อยู่จัดส่ง รวมถึงเลือกรูปแบบการจัดส่ง
  6. ระบบคำนวณราคาสินค้า ค่าจัดส่ง และยอดสุทธิที่ต้องชำระ
  7. ระบบชำระเงินผ่านเลขบัญชีและแนบหลักฐานการชำระเงิน
  8. ระบบตรวจสอบสต็อก เปรียบเทียบจำนวนสินค้า และแสดงจำนวนสินค้าคงเหลือ
  9. ระบบจองสินค้า ตัดสต็อก คืนสต็อก และอัปเดตสถานะสินค้า
  10. ระบบติดตามคำสั่งซื้อ สถานะการจัดส่ง และยืนยันการได้รับสินค้า
  11. ระบบติดต่อสอบถามหรือแจ้งปัญหาผ่านอีเมล
  12. ระบบตรวจสอบคำสั่งซื้อ ตรวจสอบการชำระเงิน และจัดเตรียมสินค้าโดยพนักงาน
  13. ระบบจัดการพนักงาน สินค้า สต็อก และการอนุมัติคำสั่งซื้อโดยแอดมิน
  14. ระบบแดชบอร์ดสรุปยอดคำสั่งซื้อ ยอดขาย สถานะการจัดส่ง และหมวดหมู่หนังสือที่ขายได้
  15. ระบบแจ้งเตือนสินค้าใกล้หมดเมื่อสินค้าแต่ละรายการเหลือน้อยกว่า 100 เล่ม

🛍️ ฝั่งลูกค้าสมาชิก (Customer)

  • เข้าสู่ระบบ
  • เข้าชมหน้ารวมสินค้า
  • ค้นหาสินค้า
  • เข้าชมรายละเอียดสินค้า
  • เพิ่มสินค้าเข้ารายการโปรด
  • รับคำแนะนำหมวดหนังสือที่เกี่ยวข้องหรือใกล้เคียง
  • เพิ่มสินค้าลงตะกร้า
  • จัดการสินค้าในตะกร้า เช่น เพิ่ม ลด หรือลบสินค้า
  • เพิ่มหรือแก้ไขที่อยู่จัดส่ง
  • เลือกรูปแบบการจัดส่ง
  • สั่งซื้อสินค้า
  • ชำระเงินผ่านเลขบัญชี
  • แนบหลักฐานการชำระเงิน
  • ดูข้อความแจ้งเตือน เช่น “คำสั่งซื้อของคุณอยู่ระหว่างการตรวจสอบ”
  • ติดตามคำสั่งซื้อและสถานะการจัดส่ง
  • ยืนยันการได้รับสินค้า
  • ดูประวัติคำสั่งซื้อของตนเอง
  • จัดการโปรไฟล์ผู้ใช้ เช่น ชื่อ รหัสผ่าน และที่อยู่จัดส่ง
  • ติดต่อสอบถามหรือแจ้งปัญหาผ่านอีเมล

👨‍💼 ฝั่งพนักงาน (Staff)

  • รับการแจ้งเตือนเมื่อมีคำสั่งซื้อใหม่
  • ตรวจสอบรายละเอียดคำสั่งซื้อ
  • ตรวจสอบหลักฐานการชำระเงิน
  • ยืนยันคำสั่งซื้อ
  • อัปเดตสถานะคำสั่งซื้อ
  • จัดเตรียมสินค้า
  • อัปเดตสถานะการจัดส่งสินค้า
  • ดูประวัติคำสั่งซื้อของลูกค้า
  • เมื่อยืนยันคำสั่งซื้อ ระบบจะตัดสต็อกอัตโนมัติ

🔐 ฝั่งผู้ดูแลระบบ (Admin)

  • จัดการพนักงาน เช่น เพิ่มหรือลดพนักงาน
  • เพิ่มสินค้าเข้าสู่หน้าร้านค้า
  • จัดการจำนวนสินค้าในสต็อก
  • อัปเดตจำนวนสินค้า เช่น เพิ่มหรือลดจำนวนสินค้า
  • ตรวจสอบสินค้าที่กำลังจะหมด
  • รับการแจ้งเตือนเมื่อสินค้าแต่ละรายการเหลือน้อยกว่า 100 เล่ม
  • ตรวจสอบยอดขาย ยอดคำสั่งซื้อ และยอดการจัดส่งผ่านกราฟต่าง ๆ
  • ตรวจสอบรายละเอียดการอนุมัติคำสั่งซื้อ
  • ตรวจสอบว่าพนักงานคนใดเป็นผู้รับผิดชอบคำสั่งซื้อนั้น

⚙️ ระบบอัตโนมัติและการประมวลผลของระบบ (System Functions)

  • เพิ่มข้อมูลบัญชีรายชื่อลูกค้าเมื่อมีการสมัครสมาชิก
  • บันทึกข้อมูลรายการโปรดของลูกค้า
  • แนะนำหมวดหนังสือที่เหมือนกันหรือเกี่ยวข้องกับสินค้าที่ลูกค้าสนใจ
  • เปรียบเทียบจำนวนสินค้าในสต็อกกับจำนวนสินค้าที่ลูกค้าสั่งซื้อ
  • คำนวณราคาสินค้าและค่าจัดส่ง เพื่อแสดงยอดสุทธิที่ต้องชำระ
  • เมื่อผู้ใช้แนบสลิปชำระเงิน ระบบจะเปลี่ยนสถานะสินค้าจาก “ว่าง” เป็น “จองสินค้า”
  • เมื่อพนักงานยืนยันคำสั่งซื้อ ระบบจะเปลี่ยนสถานะสินค้าจาก “จองสินค้า” เป็น “ขายแล้ว” และตัดสต็อกอัตโนมัติ
  • หากคำสั่งซื้อถูกยกเลิกหรือพบหลักฐานการชำระเงินไม่ถูกต้อง ระบบจะคืนสถานะสินค้าจาก “จองสินค้า” กลับเป็น “ว่าง”
  • แสดงจำนวนสินค้าคงเหลือให้ลูกค้าเห็น
  • แสดงข้อความแจ้งเตือนสถานะคำสั่งซื้อให้ลูกค้าทราบ
  • แจ้งเตือนพนักงานเมื่อมีคำสั่งซื้อใหม่เข้ามา
  • แจ้งเตือนแอดมินเมื่อสินค้าแต่ละรายการเหลือน้อยกว่า 100 เล่ม

📈 รายงานและแดชบอร์ดสำหรับแอดมิน (Reports & Dashboard)

  • สรุปยอดคำสั่งซื้อรายอาทิตย์ รายเดือน และรายปี ในรูปแบบกราฟแท่ง
  • สรุปยอดขายรายอาทิตย์ รายเดือน และรายปี ในรูปแบบกราฟเส้น
  • สรุปสถานะการจัดส่งรายวัน รายอาทิตย์ และรายเดือน ในรูปแบบกราฟแท่ง
  • สรุปหมวดหมู่หนังสือที่ขายได้รายอาทิตย์ รายเดือน และรายปี ในรูปแบบกราฟวงกลม

🔍 ปัญหาและความต้องการ | Problem Statement & Pain Points

ปัจจุบันผู้อ่านมักพบปัญหาไม่มีเวลาเดินทาง บ้านไกล หรือไปถึงร้านแล้วหนังสือหมดเพราะเช็กสต็อกล่วงหน้าไม่ได้ รวมถึงบางครั้งก็ไม่สะดวกใจที่จะเดินหาหรือสอบถามพนักงาน โครงงานนี้จึงพัฒนาระบบร้านหนังสือออนไลน์เพื่อแก้ปัญหา

เช็กสต็อกและค้นหาง่ายไม่ต้องง้อพนักงาน
มีระบบแสดงสต็อกสินค้าแบบ Real-time ให้เช็กได้จากบ้าน และใช้ระบบ Search & Filter ขั้นสูงเพื่อค้นหาหนังสือที่ต้องการได้รวดเร็ว
ช่วยตัดสินใจและสร้างความมั่นใจ
มีระบบแนะนำหนังสือสำหรับคนที่ยังไม่รู้จะซื้อเล่มไหน
สั่งซื้อง่ายและการจัดการหลังบ้านเป็นระบบ
มีระบบติดตามสถานะคำสั่งซื้อให้ลูกค้าตรวจสอบได้ตลอดเวลา และมี Dashboard ส่วนกลางให้ผู้ดูแลระบบจัดการสต็อกและออเดอร์

🗓️ ขั้นตอนการดำเนินงาน | Project Phases

ขั้นตอน (Phase) รายละเอียดโดยย่อ (Brief Description)
1. Planning กำหนดขอบเขตโครงงาน แบ่งหน้าที่ และวางแผนการดำเนินงานตลอดระยะเวลา 4 สัปดาห์
2. Analysis วิเคราะห์ความต้องการของระบบ กำหนดสิทธิ์ผู้ใช้งาน และกำหนดความสามารถหลักของระบบ
3. Design ออกแบบ Use Case Diagram, sequence diagram, โครงสร้างฐานข้อมูลและออกแบบ Wireframe
4. Development พัฒนา Frontend ด้วย React และ HTML/CSS พร้อมพัฒนา Backend API ด้วย Node.js โดยควบคุมเวอร์ชั่นโค้ดผ่าน GitHub
5. Testing ทดสอบการทำงานแบบ Manual Testing, ตรวจสอบ API ด้วย Postman และทำการทดสอบการยอมรับของผู้ใช้
6. Deployment รวบรวมระบบทั้งหมดเพื่อเตรียมสภาพแวดล้อมให้พร้อมสำหรับการสาธิต
7. Maintenance ดำเนินการแก้ไขข้อผิดพลาดที่ตรวจพบจากการทดสอบ เพื่อให้ระบบทำงานได้สมบูรณ์ตามวัตถุประสงค์ก่อนนำเสนอผลงาน

🏆 ผลลัพธ์ที่คาดว่าจะได้รับ | Expected Outcomes

  • ผู้จัดทำได้นำความรู้เชิงทฤษฎีด้านการเขียนโปรแกรมและระบบฐานข้อมูล มาประยุกต์ใช้ในการพัฒนาระบบ E-Commerce ได้อย่างเป็นรูปธรรม
  • ผู้จัดทำเกิดความเข้าใจในขั้นตอนและวงจรการพัฒนาระบบ SDLC อย่างเป็นระบบ
  • ผู้จัดทำได้รับการพัฒนาทักษะการคิดวิเคราะห์ การ Debugging และการออกแบบส่วนติดต่อผู้ใช้งาน (UI/UX)
  • ผู้จัดทำได้เรียนรู้การบริหารจัดการเวลาและการวางแผนการทำงาน เพื่อให้โครงงานสำเร็จตามกำหนดเวลา

📆 แผนการดำเนินงาน 4 สัปดาห์ | Work Plan: 4 Weeks

สัปดาห์ (Week) กิจกรรม (Activities) รายละเอียดโดยย่อ (Brief Description)
1 วิเคราะห์และออกแบบระบบ
(Analysis & Design)		 วิเคราะห์ความต้องการของระบบ โครงสร้างฐานข้อมูล ออกแบบ Use Case Diagram และ Wireframe หน้าเว็บหลัก (ทำ prototype ผ่าน Figma และเผยแพร่หน้าตัวอย่างเอง)
2 พัฒนา Frontend
(Frontend Development)		 พัฒนาหน้าเว็บด้วย HTML, CSS, JavaScript ได้แก่ หน้าแสดงรายการหนังสือ หน้ารายละเอียดสินค้า ตะกร้าสินค้า ระบบค้นหา/กรองตามหมวดหมู่ และหน้าที่จำเป็น กรอกที่อยู่จัดส่ง
3 พัฒนา Backend และฐานข้อมูล
(Backend & Database Development) 		พัฒนา API ด้วย Node.js/Express เชื่อมต่อฐานข้อมูล MySQL จัดการระบบสมาชิก ระบบจัดการสต็อกหนังสือ ระบบคำสั่งซื้อและสถานะการจัดส่ง
4 ทดสอบระบบและนำเสนอผลงาน
(Testing & Presentation)		 ทดสอบการทำงานของระบบทั้งหมด แก้ไขข้อผิดพลาด และเตรียมนำเสนอผลงาน
